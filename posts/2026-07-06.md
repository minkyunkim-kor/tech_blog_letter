## [Redis와 Valkey를 해킹해 보자(Feat. CVE ID)](https://meetup.nhncloud.com/posts/416)
- 회사: NHN
- 본문 요약: Valkey/Redis의 Lua 스크립트 오류 처리 경로에서 발견한 RESP 인젝션 취약점(CVE-2025-67733)을 분석했다. `redis.error_reply()`로 만든 오류 테이블을 `error()`로 던지면, 내부 `sdstrim()`이 문자열 양 끝의 `\r\n`만 제거하고 중간의 `\r\n`은 그대로 남겨 클라이언트 출력 버퍼에 삽입된다. 이 때문에 하나의 오류 응답이 두 개의 RESP 메시지로 쪼개져 해석되며(소켓 포이즈닝), 다음 명령의 응답이 공격자가 심은 값으로 바뀔 수 있다. 거대한 Bulk String 길이를 주입하면 클라이언트가 무한정 대기하게 만들어 DoS로도 이어질 수 있다.
- 주목할 point: `luaCallFunction → luaPushErrorBuff → addReplyErrorSdsEx → memcpy` 로 이어지는 코드 경로를 단계별로 짚어, 문자열 끝만 trim하는 사소한 로직 하나가 데이터 변조·서비스 마비로 번지는 과정을 구체적으로 보여준다. Valkey 9.0.2/8.1.6/8.0.7/7.2.12, Redis 8.4.2/8.2.5/8.0.6/7.4.8/7.2.13 이상에서 패치됨.
- keyword: #Redis #Valkey #CVE #RESP인젝션 #보안
