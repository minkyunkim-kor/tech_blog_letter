## [GitOps 하시면서, 배포 전에 뭐가 바뀔지 알고 계신가요](https://flex.team/blog/2026/07/16/backend31)
- 회사: flex
- 본문 요약: GitOps에서 PR의 values YAML diff와 실제 배포 결과(여러 values가 merge되고 Helm이 렌더한 매니페스트) 사이에 괴리가 크다는 문제를 다룬다. PR이 열리는 시점에 CI가 기준 브랜치와 PR 브랜치를 각각 Helm으로 렌더링해 ArgoCD Application 단위로 배포 diff를 계산하고, 렌더 warning/error와 함께 PR 코멘트에 노출한다. 다층 values가 merge되는 과정에서 각 값의 출처(어느 파일이 무엇을 덮었는지)도 함께 추적해 리포트로 붙인다.
- 주목할 point: 파일 diff 중심 리뷰를 배포 결과 중심 리뷰로 전환해, merge 이후 ArgoCD 화면에서 확인하던 배포 영향도를 merge 전 PR 안에서 바로 확인할 수 있도록 리뷰 시점을 앞당긴 설계.
- keyword: #GitOps #ArgoCD #Helm
