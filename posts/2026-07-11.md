## [Amazon Bedrock 모델 promptfoo 로 성능 평가하기](https://aws.amazon.com/ko/blogs/tech/bedrock-promptfoo-eval/)
- 회사: AWS
- 본문 요약: 프롬프트나 모델을 바꿀 때마다 품질이 실제로 좋아졌는지 판단하기 어려운 문제를, 오픈소스 LLM 평가 프레임워크 promptfoo로 해결하는 방법을 다룬다. Amazon Bedrock에서 제공하는 Claude Sonnet 4.6·Claude 3 Haiku·Amazon Nova Pro 세 모델을 평가 대상으로, Claude Opus 4.8을 채점용 LLM(judge)으로 두고 하나의 IAM 인증만으로 모두 호출한다. promptfooconfig.yaml 하나에 prompts·providers·tests를 선언적으로 정의해 "프롬프트 수 × 모델 수 × 테스트 케이스 수"만큼의 조합을 한 번에 실행하며, contains·latency·is-json 같은 결정적 평가와 llm-rubric 같은 모델 기반 평가를 함께 사용해 정확성·언어·구조·간결성·응답 속도를 채점한다.
- 주목할 point: LLM judge가 채점에 사용한 토큰량(19,782)이 답변 생성 토큰(2,960)의 6배를 넘었다는 실측 수치를 제시하며, 그 이유(응답 수 × rubric 수만큼 반복 채점 + 매번 답변 전문을 입력)를 명확히 짚고 judge 모델 경량화·rubric 통합·max_tokens 제한 같은 구체적인 비용 절감 방법까지 제시한 점이 실무적으로 유용하다.
