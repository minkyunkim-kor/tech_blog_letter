## [AWS Transform for migrations, 이제 13개 언어에 대한 현지화 지원](https://aws.amazon.com/ko/blogs/tech/aws-transform-for-migrations-now-supports-localization-for-13-languages/)
- 회사: AWS
- 본문 요약: AWS Transform for migrations의 콘솔 표시 언어를 한국어를 포함한 13개 언어로 확장했다는 소식이다. 검색·계획·랜딩존·네트워크 마이그레이션·리호스팅·컨테이너화 등 마이그레이션 워크플로 전반에 현지화가 적용되며, 오른쪽 상단 설정 아이콘에서 언어를 즉시 전환할 수 있다.
- 주목할 point: AWS Transform이 자연어 기반 대화형 서비스이기 때문에, 팀원이 번역 도구를 오가지 않고 자신이 사고하는 언어로 작업 계획을 검토하고 승인하고 실행할 수 있다는 점을 특히 강조했다. 현재는 마이그레이션용 AWS Transform에만 적용되고 다른 AWS Transform 제품군에는 아직 확장되지 않았다.

- 주목할 point: 예방책으로 long-term key 대신 IAM Role/STS 기반 임시 credential 사용, modelId 단위 최소 권한 부여, CloudTrail·Model Invocation Logging·CloudWatch 지표를 통한 상시 관측, Budgets·Anomaly Detection·Runbook을 통한 이상 호출 즉시 차단을 구체적인 체크리스트로 제시했다.
- keyword: #AmazonBedrock #AWS #보안 #LLMjacking

## [ag-Grid로 화면과 동일한 Excel 만들기](https://www.nextree.io/ag-gridro-hwamyeongwa-dongilhan-excel-mandeulgi/)
- 회사: 넥스트리
- 본문 요약: ag-Grid(v28.0.2)의 exportDataAsExcel()만으로는 화면과 동일한 Excel을 만들 수 없었던 이유를, valueFormatter 미적용·브라우저 CSS 미반영·헤더 스타일 미적용 세 가지 문제로 나눠 정리했다. processCellCallback으로 포맷된 텍스트를 반환하고, excelStyles와 cellClass/cellClassRules를 1:1로 매핑하는 방식으로 해결했다.
- 주목할 point: 헤더 영역은 임의의 커스텀 ID가 아니라 ag-Grid가 내부적으로 예약한 고유 ID `header`를 그대로 사용해야만 스타일이 반영된다는, 공식 문서를 뒤져야 알 수 있는 세부 사항을 짚었다. 버전(v29 이상)에 따라 Excel Export 내부 구조가 달라질 수 있다는 점도 함께 안내했다.
- keyword: #ag-Grid #Excel #프론트엔드
